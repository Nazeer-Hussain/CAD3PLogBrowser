Drag & Drop (single & multiple files)
What it does
Drag one or more log files from Explorer onto the main form to open them automatically.
Acceptance criteria
Works on form, tree panel, and log panel. Non-log files show a friendly error. Does not duplicate file-open logic.

Command-Line File Open
In Program.cs, capture command-line args and pass to the main form.
In Form_Load, if a valid file path was provided, call the existing open method.
Missing file: MessageBox with clear error. No-argument launch unchanged.

Recent Files MRU (last 10, default to PTC_LOG_DIR) [NEW]
What it does
File > Recent Files shows the last 10 opened files. File > Open defaults to the folder pointed to by the PTC_LOG_DIR environment variable.
Acceptance criteria
File > Open checks PTC_LOG_DIR env var first; falls back to last used directory if env var is absent. Recent list persists between sessions. Missing files are removed.
Commit message
File > Open: read PTC_LOG_DIR environment variable. If set and the folder exists,
  use it as the InitialDirectory for the OpenFileDialog.
  If PTC_LOG_DIR is not set, use the last used directory (saved in config.json).
Save recent files list to recentfiles.json (max 10 entries, most recent first).
Add File > 'Recent Files' submenu. Each entry opens that file.
Missing file on click: show MessageBox and remove the entry.
Add 'Clear Recent Files' at the bottom of the submenu.

A4  ?  Auto-Reload / Tail Mode
What it does
Detects when the open log file changes on disk. Shows a non-blocking status bar notification with a click-to-reload action.
Use System.IO.FileSystemWatcher on the currently open file.
On change: update a status bar label 'File updated ? click to reload' (NOT a MessageBox).
Clicking the label reloads and rebuilds the tree, preserving scroll position.
Add View > 'Watch for File Changes' toggle. Stop watcher when app closes or file changes.

A5  ?  Open Multiple Logs in Tabs
What it does
Multiple log files open simultaneously in separate tabs, each fully independent.
Add a TabControl wrapping the main split panel.
Each tab has its own tree and log panel. File > Open and drag-drop open in new tabs.
Right-click tab header: 'Close Tab', 'Close All Other Tabs'.
Tab title = filename only; tooltip = full path.

A6  ?  Merge Multiple Log Files (time-sorted) [NEW]
What it does
Load two or more log files and merge into one chronological view with source labels per line.
File > 'Merge Log Files...' opens a multi-select file dialog.
Parse each file; merge all events sorted by timestamp.
Prefix each log line with source tag: [File1], [File2], etc.
Build the call tree from the merged list. Status bar: 'Merged N files ? M lines'.

A7  ?  Compressed Log Support (.zip / .gz) [NEW]
What it does
Open .zip or .gz compressed log files directly without manual extraction.
Detect .zip and .gz extensions in the file-open logic.
For .gz: use GZipStream to decompress into MemoryStream.
For .zip: use ZipArchive. One entry ? open directly. Multiple entries ? pick dialog.
Pass decompressed content to the existing log parser unchanged.

A8  ?  Session Restore (reopen last files on startup) [NEW]
What it does
Reopens the files that were open when the app was last closed.
On app close, save open file paths to session.json (next to exe).
On startup, read session.json and reopen each file (skip missing ones).
Settings option: 'Restore last session on startup' (default: on).
Fail silently if session.json is missing or corrupt.

B3  ?  Filter by Method Name (Filter Dialog) [NEW]
What it does
Edit > Filter opens the Filter Log dialog. User builds a list of method names; the tree shows only matching nodes.
UI design
Filter Log dialog: dropdown + Add button, list of filter terms, Remove / Remove All / Filter / Close buttons. See UI Design Reference section.
Acceptance criteria
Wildcards supported. Adding a term to the list and clicking Filter applies immediately. Remove All clears the filter and shows all nodes.
Add Edit > 'Filter...' menu item (also accessible from right-click context menu).
Open the Filter Log dialog as shown in the UI Design Reference:
  - A ComboBox (dropdown) at the top with an 'Add' button
  - A ListBox below showing all active filter terms
  - Buttons: Remove (selected item), Remove All, Filter (apply), Close
When Filter is clicked: hide tree nodes whose method names are not in the list.
Show parent nodes of matching nodes to preserve hierarchy context.
When Remove All is clicked: clear the list and show all nodes.
Persist the filter list in memory for the session (not to disk).
Status bar shows: 'Filter active: N methods' when a filter is applied.

Phase 5 ? Performance Analytics  (E1 ? E6)
E1  ?  Performance Tab (Method, Calls, Time, Max, Min, Avg, Total, Self Time) [NEW]
What it does
The Performance right-panel tab shows a grid of all methods with timing statistics. 'Self Time' is the time spent within the method excluding time taken by child calls.
UI design
See Performance Tab screenshot in UI Design Reference. Columns: Method Name | No of Calls | Time | Max Time | Min Time | Average Time | Total Time | Self Time. Click column header to sort.
Self Time
Self Time = Total Time of this method ? sum of Total Time of all direct children. This isolates the method's own cost from sub-calls.
Acceptance criteria
All eight columns present. Sortable by clicking header. Clicking a row selects that method in the API Tree. Optionally dynamically updates as tree nodes are browsed (controlled by Settings toggle).
Commit message
feat: add performance tab with self-time column and sortable columns

Prompt:
Replace or enhance the existing Performance tab in the right-panel TabControl.
Use a DataGridView with these columns:
  Method Name | No of Calls | Time | Max Time | Min Time | Average Time | Total Time | Self Time
Calculate Self Time per method:
  Self Time = Total Time ? (sum of Total Time of all direct child method calls)
Click column header: sort ascending. Click again: sort descending.
Double-click a row: navigate to that method in the API Tree.
If Settings > 'Dynamically Update Performance Numbers' is checked:
  Update the Performance tab in real time as the user clicks tree nodes.
  Note: this is slow for large logs ? warn the user in the status bar if log > 50MB.
Otherwise, populate once on file open and update only on explicit refresh.
