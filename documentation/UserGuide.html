?? Introduction

CAD 3P Log Browser is a professional log file analyzer designed for analyzing performance logs from CAD applications. It provides powerful search, filtering, visualization, and export capabilities to help you quickly diagnose performance issues.

Key Features

  • ? 77 Complete Features - Comprehensive log analysis toolset
  • ? Advanced Search - Text search, regex, find all
  • ? Powerful Filtering - Multi-criteria filtering
  • ? Visualizations - Call trees, flame graphs, timelines
  • ? Performance Analysis - Detailed statistics and metrics
  • ? Multiple Export Formats - CSV, XLS, JSON, XML, Images

?? Getting Started

Opening a Log File

Method 1: File ? Open or press Ctrl+O

Method 2: Click the Open toolbar button

Method 3: Drag and drop a .log file onto the application window

?? Tip: The application remembers your last 10 opened files. Access them quickly via File ? Recent Files

Understanding the Interface

The main window is divided into several areas:

  • Menu Bar: Access to all features organized by category
  • Toolbar: Quick access to frequently used features
  • Main View: Log entries displayed as a list
  • Tab Panel: Switch between Log View, Performance, Details, Call Graph, Flame Graph, Timeline
  • Tree Panel: Call Tree and API Tree views
  • Status Bar: Shows current file, line count, selection info

?? File Operations

Opening Files

Open File: Ctrl+O or File ? Open

Opens a log file for analysis. Supports large files (500,000+ lines) efficiently using virtual list technology.

Saving & Exporting

Save Selected Lines: Ctrl+S

Saves currently selected or filtered lines to a new log file.

Export to Excel: Ctrl+Shift+E or File ? Save to XLS

Exports filtered or selected log lines to Excel format (.xls) for further analysis.

Export Performance Data: File ? Export Performance to CSV

Exports the performance statistics table to CSV format.

Auto-Reload

Reload from Disk: F5

Reloads the current file from disk. Useful when the log file is being actively updated.

?? Note: The application can automatically detect when the log file changes on disk and prompt you to reload.

?? Search & Filter

Finding Text

Find: Ctrl+F or Edit ? Find

Opens the Find dialog where you can:

  • Search for text (case-sensitive or case-insensitive)
  • Use regular expressions for advanced patterns
  • View search history

Find Next: F3 or Edit ? Find Next

Jumps to the next occurrence of the search term.

Find Previous: Shift+F3

Jumps to the previous occurrence of the search term.

Find All: Ctrl+Alt+F

Shows all matches in a separate window. Click any result to jump to that line in the main view.

Filtering

Filter Dialog: Ctrl+I or Edit ? Filter

Opens the advanced filter dialog with multiple criteria:

  • Text Filter: Show only lines containing specific text
  • Duration Filter: Show only calls that took longer than X milliseconds
  • Time Range: Filter by time of day (HH:mm:ss format)
  • Thread ID: Show only log entries from a specific thread
  • Log Level: Filter by ERROR, WARNING, INFO, DEBUG levels

All criteria are combined with AND logic (all must match).

Clear Filter: Ctrl+Shift+F or Edit ? Clear Filter

Removes the active filter and shows all log lines again.

?? Tip: When a filter is active, the status bar shows "Filter: Active" and displays the line count for both filtered and total lines.

?? Bookmarks

Managing Bookmarks

Toggle Bookmark: Ctrl+B

Adds or removes a bookmark on the currently selected line. Bookmarked lines are highlighted with a special icon.

Next Bookmark: F2

Jumps to the next bookmarked line.

Previous Bookmark: Shift+F2

Jumps to the previous bookmarked line.

Show All Bookmarks: Ctrl+Shift+B

Opens a dialog showing all bookmarked lines. Click any bookmark to jump to that line.

Clear All Bookmarks: Ctrl+Shift+Del

Removes all bookmarks from the current file.

?? Note: Bookmarks are saved automatically and will be restored when you reopen the file.

?? Navigation

Error Navigation

Next Error: F8

Jumps to the next line containing "ERROR" in the log.

Previous Error: Shift+F8

Jumps to the previous line containing "ERROR".

Next Warning: Ctrl+F8

Jumps to the next line containing "WARNING" in the log.

Previous Warning: Ctrl+Shift+F8

Jumps to the previous line containing "WARNING".

Special Navigation

Jump to Matching ENTER/EXIT: Ctrl+G

If you're on an ENTER line, jumps to the matching EXIT line (and vice versa). Very useful for analyzing method execution flow.

Jump to Line: Ctrl+L

Opens a dialog to jump directly to a specific line number.

?? Tree Views

Call Tree

Show Call Tree: Ctrl+T or View ? Show Call Tree

Displays a hierarchical tree view of method calls, showing the call stack structure. Each node shows:

  • Method name
  • Duration (color-coded: green < 100ms, amber 100-500ms, red > 500ms)
  • Call count (for repeated methods)

API Tree

Show API Tree: Ctrl+Shift+L or View ? Show API Tree

Organizes methods by API calls, grouping related functionality together.

Tree Operations

Expand All: Ctrl+E or Edit ? Expand All

Expands all nodes in the tree. For large trees, shows a progress bar and can be cancelled.

Collapse All: Ctrl+W or Edit ? Collapse All

Collapses all nodes in the tree to show only top-level items.

Tree Context Menu

Right-click on any tree node to access:

  • Copy Node Name: Copy the method name to clipboard
  • Copy Subtree: Copy the entire branch as text
  • Expand/Collapse All: Quick expand/collapse
  • Jump to Matching: Jump between ENTER and EXIT
  • Save Branch: Save this branch to a file
  • Export Branch as CSV: Export to CSV format
  • Search in Grok: Search this method in Grok (if configured)
  • Show in Other Tree: Switch between Call Tree and API Tree views

?? Performance Analysis

Performance Tab

Click the Performance tab to see a detailed statistics table with the following columns:

Column Description
Method Method name
Calls Number of times the method was called
Total (ms) Total time spent including all child calls
Avg (ms) Average time per call
Min (ms) Minimum execution time
Max (ms) Maximum execution time
Self (ms) Exclusive time (excluding child calls)
?? Tip: Click any column header to sort by that column. Click again to reverse sort order. Visual indicators (?/?) show sort direction.

Color Coding

Performance data uses color-coded heatmap visualization:

  • ?? Green: Fast execution (< 100ms)
  • ?? Amber: Moderate execution (100-500ms)
  • ?? Red: Slow execution (> 500ms) - potential bottlenecks!

?? Visualizations

Call Graph

The Call Graph tab shows a visual representation of method calls as a directed graph.

  • Nodes: Represent methods
  • Edges: Show call relationships
  • Zoom: Mouse wheel to zoom in/out
  • Pan: Click and drag to move around
  • Export: Save as image file

Flame Graph

The Flame Graph tab provides an interactive flame graph visualization perfect for identifying performance bottlenecks.

Features:

  • Interactive: Click on any method to focus on it
  • Tooltips: Hover over methods to see detailed statistics
  • Color-Coded: Colors indicate execution duration
  • Width: Proportional to time spent
  • Export: Save as image (File ? Export Flame Graph as Image)
?? Tip: The widest sections of the flame graph represent performance bottlenecks. Focus your optimization efforts there!

Timeline View

The Timeline tab shows a Gantt-chart style visualization of method execution over time.

  • Horizontal Axis: Time progression
  • Vertical Axis: Call stack depth / thread separation
  • Bars: Duration of each method call
  • Zoom/Pan: Explore specific time ranges
  • Export: Save as image file

?? Advanced Search & Filter

Regular Expression Search

In the Find dialog, check "Use Regular Expression" to enable powerful pattern matching:

  • ERROR.*timeout - Find errors related to timeout
  • \d{3,} - Find numbers with 3 or more digits
  • Thread \d+ - Find thread IDs
  • (ENTER|EXIT) - Find method boundaries

Multi-Criteria Filtering

The Filter dialog (Ctrl+I) allows you to combine multiple criteria:

Example 1: Find Slow Operations

  • Minimum Duration: 1000 ms
  • Text Contains: "Database"
  • Result: Shows only database operations taking more than 1 second

Example 2: Analyze Specific Time Window

  • From Time: 14:30:00
  • To Time: 14:35:00
  • Log Level: ERROR
  • Result: Shows only errors that occurred between 2:30 PM and 2:35 PM

?? Bookmarks Guide

What Are Bookmarks?

Bookmarks let you mark important lines in the log for quick reference. Perfect for:

  • Marking error locations
  • Highlighting key events
  • Creating a custom navigation path
  • Remembering interesting patterns

Bookmark Workflow

  1. Click on a log line to select it
  2. Press Ctrl+B to bookmark it
  3. Bookmarked line shows a bookmark icon
  4. Use F2 to jump between bookmarks
  5. Press Ctrl+Shift+B to see all bookmarks
?? Persistence: Bookmarks are saved to your settings file and will be restored when you reopen the same log file.

?? Tips & Tricks

Performance Analysis Tips

Finding Bottlenecks:
  1. Open the Performance tab
  2. Sort by "Total (ms)" column (descending)
  3. Look for methods with high total time and high call count
  4. Check the "Self (ms)" column to see exclusive time
  5. Use the Flame Graph to visualize the bottleneck
Analyzing Call Patterns:
  1. Use the Call Tree to see the call hierarchy
  2. Look for unexpectedly deep nesting
  3. Check for repeated calls that could be cached
  4. Use color coding to spot slow operations

Search Tips

Regex Examples:
  • ERROR.*\d{3,} - Errors with 3+ digit numbers
  • Thread \d+ - Find specific thread IDs
  • (timeout|failed|error) - Multiple keywords
  • \[.*ENTER.*\] - Method entry points

Filter Combinations

Example: Debug Slow Database Calls
  • Text: "Database" OR "SQL"
  • Minimum Duration: 500
  • Level: WARNING or ERROR
  • Result: Shows problematic database operations

Tree View Tips

Quick Navigation:
  • Type to search in tree (real-time filtering)
  • Right-click for context menu
  • Double-click to expand/collapse
  • Click duration to jump to that call in main view

Workflow Best Practices

Recommended Analysis Workflow:
  1. Load File: Open your log file (Ctrl+O)
  2. Quick Scan: Use F8 to jump through errors
  3. Bookmark: Mark interesting errors (Ctrl+B)
  4. Analyze Performance: Check Performance tab, sort by Total time
  5. Visualize: Use Flame Graph to identify bottlenecks
  6. Filter: Focus on problem areas (Ctrl+I)
  7. Export: Save findings for reporting

?? Troubleshooting

File Won't Open

Problem: Error when opening log file

Solutions:

  • Check file is not locked by another application
  • Verify file format is correct (.log text file)
  • Try closing and reopening the application
  • Check file permissions

Application Slow with Large Files

Problem: Performance issues with large log files

Solutions:

  • The application uses virtual list mode for efficient handling of 500,000+ lines
  • Use filters to reduce displayed data (Ctrl+I)
  • Close tree views if not needed (they consume memory)
  • Increase allocated memory in Settings if available

Search Not Finding Results

Problem: Search doesn't find expected results

Solutions:

  • Check case sensitivity setting
  • Verify regex syntax if using regular expressions
  • Ensure no active filter is hiding results
  • Try "Find All" to see all matches

Bookmarks Not Saved

Problem: Bookmarks disappear when reopening file

Solutions:

  • Check %AppData%\CAD3PLogBrowser\ folder exists and is writable
  • Verify appsettings.json is not read-only
  • Ensure application has write permissions

? Frequently Asked Questions

What log file formats are supported?

The application supports plain text log files (.log, .txt) with timestamped entries. It auto-detects common log formats and parses ENTER/EXIT markers for method tracing.

How large of a file can I analyze?

The application uses virtual list mode and can efficiently handle files with 500,000+ lines. Performance may vary based on available system memory.

Can I analyze multiple files at once?

Currently, the application analyzes one file at a time. However, you can export results and compare them manually. Multiple instance support may be added in future versions.

Where are settings stored?

Settings are stored in JSON format at: %AppData%\CAD3PLogBrowser\appsettings.json

This includes window state, recent files, bookmarks, preferences, and more.

How do I reset to default settings?

Delete the appsettings.json file from %AppData%\CAD3PLogBrowser\ and restart the application. Default settings will be recreated automatically.

Does it support multiple languages?

The application is fully localization-ready. Language support can be added by creating translated resource files (Resources.{language}.resx).

?? Advanced Features

Grok Integration

If you have Grok configured, you can right-click on any method in the tree view and select "Search in Grok" to search for that method name in your Grok instance.

Branch Operations

In tree views, you can work with specific branches:

  • Save Branch: Right-click node ? Save Branch (saves that node and all children)
  • Export Branch: Right-click node ? Export Branch as CSV
  • Navigate Branch: Right-click node ? Jump to matching ENTER/EXIT

Custom Highlighting

Define keywords in Settings ? General ? Keywords. These keywords will be highlighted in the log view for easy spotting.

Performance Thresholds

Customize what constitutes "slow" execution in Settings ? Performance:

  • Green threshold: Default < 100ms
  • Amber threshold: Default 100-500ms
  • Red threshold: Default > 500ms
