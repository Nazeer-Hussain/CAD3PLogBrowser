?? Getting Started

Opening Log Files

  1. Click File ? Open (or press Ctrl+O)
  2. Navigate to your log file (usually .log or .log.* extension)
  3. Select the file and click Open
  4. Wait for the file to load (progress shown in status bar)
?? Tip: Set the PTC_LOG_DIR environment variable to your log directory for quick access!

Understanding the Interface

The main window consists of several key areas:

Area Location Purpose

Tab Panel Right side Log, Performance, Graphs, Timeline
Status Bar Bottom File info, line count, selection details

Your First Analysis

Follow these steps for a typical analysis workflow:

  1. Open a log file using File ? Open or Ctrl+O
  2. Review the Call Tree (left panel) to see the API call hierarchy
  3. Check for errors by clicking the "?? Error" button in toolbar
  4. Analyze performance in the Performance tab to find slow operations
  5. Use Find/Filter to narrow down to specific areas
  6. Bookmark important lines for future reference
  7. Export results if needed (CSV, JSON, images)

?? Search & Filter

Basic Search

Find text in your log files:

  1. Press Ctrl+F or click the Find button
  2. Enter your search text
  3. Choose options: Match case, Whole word, Regex
  4. Click Find Next or press Enter
  5. Use F3 to find next occurrence

Regular Expression Search

Use powerful regex patterns for advanced searching:

Pattern Description Example
\d+ Match numbers Find all line numbers
ERROR|EXCEPTION Match either pattern Find errors or exceptions
^\d{4}-\d{2}-\d{2} Match date format Find timestamps
\[(\d+)\s*ms\] Capture duration Find performance timings

Advanced Filtering

The Filter dialog (Ctrl+I) provides comprehensive filtering:

Text Filter

  • Filter by text content (with regex support)
  • Case-sensitive option
  • Whole word matching

Duration Filter

  • Filter API calls by minimum duration
  • Find slow operations (e.g., >100ms)
  • Useful for performance analysis

Time Range Filter

  • Filter logs by time range
  • Focus on specific time windows
  • Great for isolating incidents

Thread & Level Filter

  • Filter by thread ID
  • Filter by log level (Error, Warning, Info, Debug)
  • Combine multiple criteria
?? Pro Tip: Combine multiple filter criteria for precise log analysis. For example, filter for ERROR level + duration >50ms + specific thread ID.

?? Tree Views

Call Tree

The Call Tree shows the hierarchical structure of API calls:

  • Green ?: ENTER/EXIT matched correctly
  • Red ?: Missing EXIT (potential issue)
  • Duration: Shown in milliseconds [XX ms]
  • Color coding:
    • Green: Fast (<100ms)
    • Orange: Medium (100-500ms)
    • Red: Slow (>500ms)

Tree Operations

  • Ctrl+E - Expand all nodes
  • Ctrl+W - Collapse all nodes
  • Ctrl+G - Jump to matching ENTER/EXIT
  • Right-click for context menu (copy, export, search)

API Tree

The API Tree groups all invocations of each API:

  • Organized by API name
  • Shows call count per API
  • Lists all invocation line numbers
  • Sortable by name, count, or first occurrence

Tree Search NEW

The search box above the tree views filters nodes in real-time:

  1. Click in the search box above the tree
  2. Type your search text (e.g., API name)
  3. Tree automatically filters to show matching nodes
  4. Clear the search box to show all nodes again
?? New in v2.5: Tree search box is now properly positioned and visible in both light and dark themes!
