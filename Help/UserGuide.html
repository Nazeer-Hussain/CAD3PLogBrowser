?? Introduction

CAD 3P Log Browser is a professional log file analyzer designed specifically for analyzing performance logs from CAD applications. It provides powerful search, filtering, visualization, and export capabilities to help developers and support teams quickly diagnose performance issues and understand application behavior.

?? IMAGE NEEDED: Main application window showing all key areas
Filename: main-window.png - Overview of CAD 3P Log Browser interface

Key Features

  • ? 77 Complete Features - Comprehensive log analysis toolset
  • ? Advanced Search - Text search with regex support
  • ? Powerful Filtering - Multi-criteria filtering (duration, time, thread, level)
  • ? Rich Visualizations - Call trees, flame graphs, timelines, call graphs
  • ? Performance Analysis - Detailed statistics with color-coded heatmaps
  • ? Multiple Export Formats - CSV, XLS, JSON, XML, images
  • ? Bookmarks - Mark important lines for quick reference
  • ? Large File Support - Efficiently handles 500,000+ lines

Who Should Use This?

  • Developers: Debug performance issues and understand call flows
  • QA Engineers: Analyze test results and identify bottlenecks
  • Support Teams: Diagnose customer issues from log files
  • Performance Engineers: Profile application behavior and optimize code

Regular Expression Search

Enable powerful pattern matching by checking "Use Regular Expression" in the Find dialog.

Common Regex Patterns:

Pattern Description Example
ERROR.*timeout Find errors related to timeout Matches "ERROR: Connection timeout"
\d{3,} Find numbers with 3+ digits Matches "1000", "52347"
Thread \d+ Find thread IDs Matches "Thread 1234"
(ENTER|EXIT) Find method boundaries Matches "ENTER" or "EXIT"
\[.*ERROR.*\] Find bracketed errors Matches "[ERROR: Failed]"
Use the regex tester at regex101.com to test your patterns before using them in the application.

Filter Criteria:

Criterion Description Example
Text Filter Show only lines containing specific text "Database" shows only database-related lines
Duration Filter Show only calls taking longer than X milliseconds "1000" shows calls taking > 1 second

?? Performance Analysis

The Performance tab provides detailed statistics about method execution times, helping you identify bottlenecks and understand application behavior.

?? IMAGE NEEDED: Performance tab showing statistics table with color-coded values
Filename: performance-tab.png

Performance Statistics Table

Click the Performance tab at the bottom of the window to view the statistics table.

Columns Explained:

Column Description What to Look For
Method Name of the method Identify specific methods causing issues
Calls Number of times the method was called High call counts may indicate inefficiency
Total (ms) Total time spent including all child calls Methods with highest total time are good optimization candidates
Avg (ms) Average time per call Inconsistent averages may indicate variable performance
Min (ms) Minimum (fastest) execution time Best-case performance
Max (ms) Maximum (slowest) execution time Worst-case performance - outliers may indicate issues
Self (ms) Exclusive time (excluding child calls) High self-time indicates the method itself is slow, not its children

Sorting the Table

Click any column header to sort by that column. Click again to reverse the sort order.

Visual indicators (? ascending, ? descending) show the current sort column and direction.

Useful Sort Strategies:

  • Sort by Total (descending): Find the most time-consuming operations overall
  • Sort by Calls (descending): Find frequently-called methods that may benefit from caching
  • Sort by Self (descending): Find methods that are intrinsically slow
  • Sort by Max (descending): Find methods with worst-case performance issues

Color-Coded Heatmap

Performance data uses color-coded visualization to quickly identify problem areas:

Color Duration Range Meaning
?? Green < 100ms Fast execution - no issues
?? Amber 100ms - 500ms Moderate execution - monitor if frequent
?? Red > 500ms Slow execution - potential bottleneck! Investigate these first
?? IMAGE NEEDED: Performance table with color-coded cells showing heatmap
Filename: performance-heatmap.png
You can customize the color thresholds in Settings ? Performance ? Thresholds

Finding Bottlenecks - Step by Step

  1. Click the Performance tab
  2. Click the Total (ms) column header to sort by total time (descending)
  3. Look for red-highlighted methods at the top of the list
  4. Check the Self (ms) column - high self-time means the method itself is slow
  5. Check the Calls column - many calls × moderate time = bottleneck
  6. Right-click the method and select "Show in Call Tree" to see its context
  7. Use the Flame Graph tab for visual confirmation
Focus optimization efforts on methods that appear in the top 10% of total time AND have high self-time. These are your biggest bottlenecks.

?? Visualizations

CAD 3P Log Browser provides three powerful visualizations to help you understand performance and call relationships.

Flame Graph

The Flame Graph provides an interactive visualization perfect for identifying performance bottlenecks at a glance.

?? IMAGE NEEDED: Flame Graph showing method call hierarchy with color-coded bars
Filename: flame-graph.png

How to Use the Flame Graph:

  1. Click the Flame Graph tab
  2. View the hierarchical visualization where:
    • Width = Time spent (wider = more time)
    • Height = Call stack depth
    • Color = Duration indicator (green/amber/red)
  3. Click any method to zoom in and focus on that subtree
  4. Hover over methods to see detailed statistics in tooltips
  5. Look for the widest sections - these are your bottlenecks!
The widest sections at any level represent where the most time is spent. These are your primary optimization targets.

Flame Graph Features:

  • Interactive: Click to focus on specific method subtrees
  • Tooltips: Hover for method name, duration, call count
  • Color-Coded: Quick visual identification of slow operations
  • Proportional: Width accurately represents time distribution
  • Export: Save as image via File ? Export Flame Graph as Image

Timeline View

The Timeline tab shows a Gantt-chart style visualization of method execution over time.

?? IMAGE NEEDED: Timeline view showing method executions as horizontal bars over time
Filename: timeline-view.png

Timeline Features:

  • Horizontal Axis: Time progression from left to right
  • Vertical Axis: Call stack depth or thread separation
  • Bars: Each bar represents a method call, length = duration
  • Colors: Different colors distinguish between methods
  • Zoom: Mouse wheel to zoom in/out on time ranges
  • Pan: Click and drag to move through timeline
  • Export: Save as image via File ? Export Timeline as Image

What to Look For in Timeline:

  • Long bars: Methods taking significant time
  • Gaps: Idle time or synchronization delays
  • Overlaps: Concurrent operations (multi-threading)
  • Patterns: Repeated sequences that might be optimized

?? Common Workflows

Workflow 1: Quick Error Analysis

Goal: Quickly find and analyze all errors in a log file

  1. Open log file (Ctrl+O)
  2. Press F8 to jump to first error
  3. Review the error and surrounding context
  4. Press Ctrl+B to bookmark important errors
  5. Continue pressing F8 to find remaining errors
  6. Press Ctrl+Shift+B to view all bookmarked errors
  7. Export bookmarked lines (Ctrl+S) for reporting

Workflow 2: Performance Bottleneck Analysis

Goal: Identify and document performance bottlenecks

  1. Open log file with performance data
  2. Click Performance tab
  3. Sort by Total (ms) column (descending)
  4. Identify top 5 slowest methods (look for red highlighting)
  5. Check Self (ms) to see if method itself is slow
  6. Switch to Flame Graph tab for visual confirmation
  7. Note the widest sections - these are your bottlenecks
  8. Export Flame Graph as image for documentation
  9. Use Call Tree to understand call hierarchy
  10. Export performance data to CSV for detailed reporting

Workflow 3: Thread-Specific Analysis

Goal: Analyze behavior of a specific thread

  1. Open log file
  2. Press Ctrl+I to open filter dialog
  3. Enter thread ID in "Thread ID" field (e.g., "Thread 1234")
  4. Click "Apply" - now viewing only that thread's activities
  5. Analyze the filtered results
  6. Use Timeline tab to visualize thread activity over time
  7. Export filtered results if needed
  8. Press Ctrl+Shift+F to clear filter when done

Workflow 4: Time Window Investigation

Goal: Investigate what happened during a specific time period

  1. Open log file
  2. Press Ctrl+I for filter dialog
  3. Set "From Time" and "To Time" (e.g., 14:30:00 to 14:35:00)
  4. Optionally add level filter (e.g., ERROR or WARNING)
  5. Click "Apply"
  6. Review filtered results
  7. Use bookmarks to mark key events in this time window
  8. Export to Excel for detailed analysis

?? Tips & Tricks

Performance Analysis Tips

Finding True Bottlenecks: A method with high Total time but low Self time is calling expensive child methods. Focus on optimizing the children, not the parent.
Frequent Small Calls: A method called 10,000 times taking 50ms each is a bigger problem than one called once taking 100ms. Sort by Calls × Avg to find these.
Use Multiple Views: Cross-reference findings across Performance table, Flame Graph, and Timeline for comprehensive understanding.

Search Tips

Regex Power: Use (error|fail|timeout) to find multiple keywords at once.
Case Sensitivity: Disable case sensitivity when searching for ERROR, Error, and error variations.

Tree View Tips

Quick Find in Tree: Type to search - the tree filters in real-time as you type.
Compare Trees: Use "Show in Other Tree" context menu to see the same method in both Call Tree and API Tree perspectives.

Bookmark Tips

Strategic Bookmarking: Bookmark the first and last occurrence of an issue to quickly measure its duration and scope.
Color-Code with Keywords: Add keywords in Settings to visually highlight bookmarked concepts.

Export Tips

Filtered Excel Export: Apply a filter first, then export to Excel to get only the relevant data in your spreadsheet.
High-Res Images: Exported visualization images are high-resolution and suitable for professional presentations.

?? Troubleshooting

File Won't Open

Symptom: Error message when trying to open a log file

Solutions:

  • Check file is not locked by another application (close the log-generating application)
  • Verify file format is plain text (.log or .txt)
  • Check file permissions - ensure you have read access
  • Try closing and reopening CAD 3P Log Browser
  • If file is very large (>1GB), it may take time to load - be patient

Application Slow with Large Files

Symptom: Performance issues when analyzing large log files

Solutions:

  • Use filters (Ctrl+I) to reduce displayed data
  • Close tree views if not needed (they consume memory for large datasets)
  • Collapse tree nodes you're not investigating
  • Close other tabs (Flame Graph, Timeline) when not in use
  • Increase available system memory if possible
The application uses virtual list mode and can handle 500,000+ lines, but tree views require building complete data structures in memory.

Search Not Finding Results

Symptom: Search doesn't find expected results

Solutions:

  • Check case sensitivity setting in Find dialog
  • If using regex, verify syntax at regex101.com
  • Ensure no active filter is hiding results - press Ctrl+Shift+F to clear
  • Try "Find All" to see all matches at once
  • Check you're searching in the correct direction (forward vs backward)

Bookmarks Not Saved

Symptom: Bookmarks disappear when reopening the file

Solutions:

  • Check %AppData%\CAD3PLogBrowser\ folder exists and is writable
  • Verify appsettings.json is not read-only
  • Ensure application has write permissions to AppData folder
  • Try running the application as administrator (one time) to create necessary folders

Visualizations Not Showing

Symptom: Flame Graph, Timeline, or Call Graph tabs are empty

Solutions:

  • Ensure log file has proper ENTER/EXIT markers for method calls
  • Check that duration information is present in log entries
  • Some visualizations require specific log format - verify your log format is supported
  • Try with a known-good sample log file to rule out file format issues

Help File Doesn't Open

Symptom: Pressing F1 shows error or does nothing

Solutions:

  • Verify UserGuide.html exists in Help\ subfolder or application directory
  • Check default browser is configured correctly in Windows
  • Try opening UserGuide.html manually from Windows Explorer
  • As fallback, use Ctrl+K for keyboard shortcuts dialog

? Frequently Asked Questions

General Questions

Q: What log file formats are supported?

A: CAD 3P Log Browser supports plain text log files (.log, .txt) with timestamped entries. It auto-detects common log formats and parses ENTER/EXIT markers for method tracing.

Q: How large of a file can I analyze?

A: The application uses virtual list mode and can efficiently handle files with 500,000+ lines. Performance depends on available system memory, especially when using tree views and visualizations.

Q: Can I analyze multiple files at once?

A: Currently, the application analyzes one file at a time. However, you can run multiple instances of the application to analyze different files simultaneously.

Settings & Configuration

Q: Where are settings stored?

A: Settings are stored in JSON format at: %AppData%\CAD3PLogBrowser\appsettings.json

This includes window state, recent files, bookmarks, preferences, and more.

Q: How do I reset to default settings?

A: Delete the appsettings.json file from %AppData%\CAD3PLogBrowser\ and restart the application. Default settings will be recreated automatically.

Features & Capabilities

Q: Does it support regex in search?

A: Yes! Check "Use Regular Expression" in the Find dialog (Ctrl+F) to enable powerful pattern matching.

Q: Can I export filtered results?

A: Yes! Apply your filter first, then export using Ctrl+Shift+E for Excel or Ctrl+S for text format. Only filtered lines are exported.

Q: Does it support multiple languages?

A: The application is fully localization-ready. Additional languages can be added by creating translated resource files.

Performance & Optimization

Q: Why is tree view slow with my large log?

A: Tree views build complete hierarchical structures in memory. For very large logs (100,000+ method calls), this requires significant memory. Use filters to reduce data size or close tree views when not needed.

Q: How can I speed up analysis?

A: Tips for faster analysis:

  • Close unused tabs (Flame Graph, Timeline, Call Graph)
  • Use filters to reduce dataset size
  • Collapse tree nodes you're not investigating
  • Disable file monitoring if not needed
  • Increase available system RAM

Troubleshooting

Q: Why can't I open my log file?

A: Common causes:

  • File is locked by another application
  • File is not plain text format
  • Insufficient permissions to read file
  • File path contains special characters

Q: My bookmarks disappeared - why?

A: Bookmarks are file-specific and saved by file path. If you:

  • Moved or renamed the log file
  • Opened from different path/location
  • Settings file was deleted/corrupted

The bookmarks won't be found. Always open files from the same location.
