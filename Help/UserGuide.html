📚 CAD 3P Log Browser

Complete User Guide & Reference Manual

v2.7 .NET Framework 4.8 Windows

📚 Introduction

CAD 3P Log Browser is a professional log-file analyser built specifically for CAD application trace logs. It parses structured ENTER/EXIT markers, computes precise per-method timings, and gives you multiple interactive views — all inside a fast, virtual-mode UI that handles 500,000+ line files without breaking a sweat.

🌳 Analysis Views

  • Call Tree (hierarchical call stack)
  • API Tree (grouped by method)
  • Raw View (unformatted file text)
  • Performance statistics table
  • Call Graph (dependency graph)
  • Flame Graph (profiling view)
  • Timeline (Gantt-style view)
  • AI Assistant

📊 Key Capabilities

  • Virtual-mode log view (500 k+ lines)
  • Regex / multi-criteria filtering
  • Bookmarks with persistence
  • Error / Warning navigation
  • Dark & Light themes
  • Export to CSV, JSON, XML, PNG, XLS
  • Merge multiple log files
  • Lazy loading for huge trees

⭐ What’s New in v2.7

📝 Raw View Tab

A new Raw tab shows the unformatted file content in a read-only, scrollable text box (Consolas font, horizontal + vertical scrollbars). Files above 50,000 lines are capped with a visible notice. Toggle it via View → Show Tabs → Raw.

📄 API Tree — ENTER/EXIT counting fixed

Each invocation of an API now appears as exactly one node, regardless of whether ENTER+EXIT, ENTER-only, or EXIT-only exists. Orphan EXIT lines (no matching ENTER) are shown with an [EXIT only] label and a red ❌ icon. Call counts are no longer doubled.

⚙ Settings Dialog — fully redesigned

The settings dialog is now a compact TabControl with six organised pages instead of a single long scroll. A Reset to Defaults button (bottom-left) instantly resets all non-security settings. New settings added: Default open folder, Default tree view, and Start-up tab.

🌲 Start-up tab & tree view restored

The Start-up tab and Default tree view settings are now correctly applied when the application launches, not just saved. Choose any of the 8 tabs and Call Tree or API Tree as your preferred opening state.

📅 Show Tabs menu — all 8 tabs, correct labels & icons

The View → Show Tabs submenu now contains all eight tabs with meaningful names (Log View, Raw, Performance, Log Details, Call Graph, Flame Graph, Timeline, AI Assistant) and matching icons. Previously, Call Graph had no entry and several items had wrong labels.

🎨 Theme switching performance improved

Switching themes no longer re-subscribes drawing handlers on every control on every call — only the first switch from light↔dark direction incurs the full walk. Repeated theme toggles are now near-instant.

🚫 Expand/Collapse icons corrected

The Expand All and Collapse All toolbar buttons now use the correct Segoe MDL2 glyphs (ChevronDown / ChevronRight) instead of the generic Add/Remove symbols that rendered as ?? on some systems.

🔧 All toolbar icons render on Windows 10 RTM+

Eleven icon codepoints that were in the extended PUA range (E9xx–EDxx) and unavailable on older Windows 10 builds have been replaced with equivalents in the guaranteed-available E7xx–E8xx range.

📁 Resource string encoding fixed

Six resource strings (sort labels, dialog titles, summary row) were double-encoded as mojibake. They now display correctly: ↓, —, ─, ✓, ✗.

🚀 Getting Started

Opening Your First Log File

📁 File → Open Press Ctrl+O, click the Open toolbar button, or drag & drop a .log file onto the window.
🕐 Recent Files The last 10 files appear under File → Recent Files. Click any entry to reopen immediately.
🔄 Merge Logs Press Ctrl+M to open multiple files and merge them into a single time-sorted view.
💡 Tip: Set the environment variable PTC_LOG_DIR to your log directory. The Open dialog will default to that folder automatically.

Recommended First Steps

  1. Open a log file (Ctrl+O).
  2. Press F8 a few times to scan through errors.
  3. Switch to the Performance tab and sort by Total (ms) to find the slowest methods.
  4. Click any tree node on the left to jump to that line in the log.
  5. Use Ctrl+I to filter the log down to interesting lines.

🌳 Call Tree

The Call Tree shows method invocations as a nested hierarchy, exactly mirroring the call stack at runtime. Activate it with the Call Tree toolbar button or View → Show Call Tree.

Node Format

Node textMethodName [142 ms]  or  MethodName [<1 ms]  or  MethodName [? ms] (unmatched)
✅ icon (green check)Matched ENTER + EXIT pair found
❌ icon (red cross)No matching EXIT found for this ENTER
TooltipShows method name, source file, enter line, exit line, duration

Duration Colour Coding

ColourDurationMeaning
Green< 100 msFast — no concern
Amber100 – 500 msModerate — worth monitoring
Red> 500 msSlow — likely bottleneck

Lazy Loading

For very large call trees (> 50,000 nodes), the tree uses lazy loading: child nodes are populated on demand when you expand a parent. A “Loading…” placeholder is shown until children are rendered.

Sorting (API Tree only)

Right-click the API Tree to choose sort order: by Name, by Call Count, or by First Line.

🔌 API Tree

The API Tree groups all invocations of each method under a single root node, showing call count in brackets. This makes it easy to see how many times each API was called and navigate between individual invocations.

Root node formatMethodName (N calls) — click to scroll to first occurrence
Child node (normal)MethodName — Ln 1234 — ENTER line; click to scroll to that invocation
Child node (orphan)MethodName — Ln 1234  [EXIT only] — EXIT with no matching ENTER; always shown with ❌
✅ icon (green check)All invocations have a matching ENTER + EXIT pair
❌ icon (red cross)One or more ENTER lines lack a matching EXIT (or vice versa)
One node per callEach invocation appears as exactly one child node whether ENTER+EXIT, ENTER-only, or EXIT-only exists

Selecting an API Tree node also displays its invocation details in the Log Details tab (API name, total count, first occurrence, all line numbers, match status).

🔎 Tree Search

The text box at the top of the tree panel filters nodes in real time as you type.

  • Nodes whose text matches are highlighted in yellow.
  • Parent nodes that contain a match are automatically expanded and shown with a transparent background.
  • Non-matching leaf nodes are collapsed.
  • Clear the box to restore the full tree.
💡 Tip: The search is case-insensitive and matches any substring of the node label, including the duration suffix (e.g. typing 142 finds nodes labelled [142 ms]).

💡 Tips & Best Practices

Finding Performance Bottlenecks

  1. Open the Performance tab.
  2. Click Total (ms) to sort descending.
  3. The top rows are your most expensive methods — focus there first.
  4. Check Self (ms): a high Self value means the method itself is slow, not its children.
  5. Switch to the Flame Graph to see the call-stack context of the bottleneck visually.

Investigating an Error

  1. Press F8 to jump to the first error.
  2. Press Ctrl+B to bookmark it.
  3. Press Ctrl+G if the error is inside a method call to jump to the surrounding ENTER/EXIT context.
  4. Right-click the enclosing tree node and choose Save Branch to extract just that call’s log lines for a bug report.

Narrowing Down a Large Log

  1. Press Ctrl+I to open the Filter dialog.
  2. Add a Time Range to focus on the incident window.
  3. Add a Text filter for a relevant module or method name.
  4. Combine with a Log Level filter of ERROR or WARNING.
  5. The status bar shows how many lines match; iterate until the count is manageable.

Cross-Referencing Trees

  • Select a node in the Call Tree → right-click → Show in API Tree to see all occurrences of that method.
  • Select a node in the API Tree → right-click → Show in Call Tree to see one occurrence in full call-stack context.

Using Bookmarks Effectively

  • Mark the start and end of a slow period with Ctrl+B.
  • Use F2 / Shift+F2 to cycle through just those anchors.
  • Bookmarks survive file reloads — you can bookmark, reload, and continue where you left off.

🛠 Troubleshooting

File won’t open

Symptoms: Error dialog on open, or file shows 0 lines.
Fixes: Check file is not locked by another process. Verify it is a plain-text file (not binary). Check read permissions. Try copy-pasting the file first.

Application is slow with a very large file

Note: Virtual-mode rendering means the log view itself should always be fast. Slowness usually comes from building the call tree for logs with millions of API calls. Use Ctrl+I to filter the log before loading trees, or enable lazy loading (automatic for > 50,000 nodes).

Tree is empty after loading

  • The parser looks for lines containing [ENTER] and [EXIT] markers. If your log format uses different markers, the tree will be empty.
  • Check that the log is the raw trace output and not a filtered/truncated copy.

Bookmarks not restored on reopen

  • Bookmarks are keyed to the full file path. If you moved or renamed the file, bookmarks are not found.
  • Check that %AppData%\CAD3PLogBrowser\ exists and is writable.

Theme toggle not working

  • Press Ctrl+T — if it works via keyboard but not the button, the toolbar icon may not have been created yet (check Settings for icon size).
  • Try opening Settings and re-selecting the theme there.

AI Assistant returns no results

  • In built-in mode: a file must be loaded first.
  • In Claude mode: verify your API key in Settings, check internet connectivity, and confirm the key has sufficient quota.

Expand All hangs

  • For very large trees, Expand All may take time. A progress message appears in the status bar.
  • Press Esc or click the status bar to cancel the operation.

Resetting to defaults

Delete %AppData%\CAD3PLogBrowser\appsettings.json and restart. All settings (window state, recent files, bookmarks, theme, font) will be recreated with defaults.

❓ FAQ

What log file formats are supported?

Any plain-text file. The parser auto-detects timestamp format and looks for [ENTER] / [EXIT] markers. Supported extensions: .log, .log.*, .* (any text file).

How large a file can I open?

The virtual-mode list view handles files with 500,000+ lines smoothly. Call-tree building becomes slower above ~200,000 API calls; use filtering first to reduce scope.

Can I open multiple files simultaneously?

One file at a time per window. Use File → Merge Log Files… to combine multiple files into a single time-sorted view within one window.

Where are my settings saved?

%AppData%\CAD3PLogBrowser\appsettings.json

How do I configure the default open directory?

Set the PTC_LOG_DIR environment variable, or set a custom path in Options → Settings → Initial Directory. The env variable takes precedence unless a custom path is also set.

Can I use the Flame Graph without an internet connection?

Yes. The Flame Graph, Timeline, Call Graph, and all other visualisations are rendered locally with no network access required. Only the AI Claude mode and the “Check for Updates” / “Report Errors” links require internet.

Where do I report bugs or request features?

Use Help → Report Errors to open the GitHub Issues page, or visit github.com/Nazeer-Hussain/CAD3PLogBrowser/issues.
